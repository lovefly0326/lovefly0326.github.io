odoo Tree

发表于 2021-03-04 更新于 2021-03-15 分类于 odoo12 阅读次数：

Tree

tree 视图属性

editable

tree 视图可直接编辑

  • top 新建记录在列表顶部

  • bottom 新建记录在列表底部

default_order

根据给定顺序字段的值进行排序

decoration-{$name}

允许基于相应记录的属性更改行文本的样式，值是 Python 表达式

  • decoration-bf 字体加粗，相当于font-weight: bold

  • decoration-it 斜体，相当于font-style: italic

  • $name 颜色相关(danger, info, muted, primary, success, warning)

create、delete

禁止创建和删除记录(即隐藏创建和删除按钮)

limit

默认一页的数量(正整数)

tree 视图可包含的子元素

button

在tree视图列表单元格中显示按钮，执行特定的动作

icon

按钮图标

  • 以fa-开头，会渲染成i标签，使用 Font Awsome 图标库的图标

  • 图片相对路径，渲染成img标签

string

  • 没有设置icon，显示为文本按钮

  • 设置了icon，只显示图标，string 作为按钮的title属性

  • type

    • object 使用当前记录的 id 和当前的上下文，调用列表模型上的name对应的方法

    • action 加载name对应的 action，上下文扩展为 list 的模型(active_model)，当前行的记录(active_id)和 list 中所有已加载的记录(active_ids)

name

参照 type

args

传递给name方法的参数（列表），不常用

attrs

  • invisible 一个 domain 条件，在当前记录的上下文中求值，为 True 则隐藏，为 False 则显示

states

  • invisible 的简写，值列表通过逗号分隔。要求模型中有state 字段，并且在视图中使用该字段。

  • state 字段的值如果在states值列表中，则隐藏 button

  • 注意点：states和attrs一起使用可能会导致意外的结果，因为 domain 通过 AND 连接

context

执行按钮的调用动作时会合并到视图的上下文中

class

样式类，可自定义类，也可以直接使用bootstrap 类

field

tree视图每列显示的字段

name

当前模型中定义的字段名称。同一个name每个视图中只能使用一次

string

  • 列标题默认使用模型字段的string
  • 可通过 field 属性string 覆盖模型中的 string

invisible

  • 获取并储存，但是不显示在表格中的字段

  • 对于需要被使用，但不需要显示的字段是必须的。比如decoration-{$name}表达式中依赖的字段

groups

权限组(可以看到该字段的权限组，多个以逗号隔开)

widget

字段挂件（将字段以特定的方式展示），具体参考 widget

sum、avg、max、min

列汇总求和、平均值、最大值、最小值

attrs

基于记录值的动态属性，只会作于当前字段

  • invisible 当前记录的当前字段列是否隐藏（不会隐藏列）

  • column_invisible 常用于根据父对象隐藏列

<control>

为当前视图定义自定义控件(one2many)。不支持任何属性，可以有子级(<create> )。
不自定义 control 时，默认值是Add a line(添加明细行)
只会在编辑状态且 create=”1”(默认就为 1)的情况下生效

<create>

添加按钮以在当前列表中创建新元素

  • string 按钮名称
  • context 检索新记录的默认值时，将合并到现有上下文中。例如覆盖默认值
