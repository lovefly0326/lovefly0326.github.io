字段挂件

发表于 2021-09-23 更新于 2021-09-27 分类于 odoo12

odoo12 原生字段挂件

integer (FieldInteger)

整型（integer）字段的默认字段类型
支持的字段类型: integer

  • options
    • type: 输入类型，支持text（默认）、number（存在浏览器兼容性问题）
    • step: 步长（只支持number类型）

handle (HandleWidget)

作用于列表视图显示为一个手柄，用于拖拽排序
支持的字段类型: integer
不支持同一列表中配置多个handle

float (FieldFloat)

浮点型（float）字段的默认字段类型
支持的字段类型: float

  • Attributes

    • digits: 精度配置

  • options

    • type: 输入类型，支持text（默认）、number（存在浏览器兼容性问题）
    • step: 步长（只支持number类型）

float_time (FieldFloatTime)

将浮点数显示为时间间隔（小时为单位）
支持的字段类型: float

float_factor (FieldFloatFactor)

用于显示通过给定factor转换后的值
支持的字段类型: float

  • options
    • factor: 转换因子(默认为1)。界面显示值 = 数据库实际值 * 转换因子

float_toggle (FieldFloatToggle)

用于在给定值列表中切换数据
支持的字段类型: float

  • options
    • factor: 同float_factor
    • range: 切换的值列表(经过factor转换后的值)。默认为[0.0, 0.5, 1.0];

monetary (FieldMonetary)

货币型（monetary）字段的默认字段类型
支持的字段类型: monetary、float

  • options
    • currency_field: 币种对应字段

boolean (FieldBoolean)

布尔型（boolean）字段的默认字段类型
支持的字段类型: boolean

boolean_favorite (FavoriteWidget)

根据布尔字段的值，显示为空(或非空)的星星
支持的字段类型: boolean
注意：该挂件在readonly状态下仍然可以编辑


boolean_toggle (BooleanToggle)

根据布尔字段的值，显示为切换开关
支持的字段类型: boolean
注意：该挂件在readonly状态下仍然可以编辑

toggle_button (FieldToggleBoolean)

显示为一个圆点（绿色代表true，灰色代表false）
支持的字段类型: boolean
注意：该挂件在readonly状态下仍然可以编辑

  • options
    • active: 选中时的tooltip提示
    • inactive: 选中时的tooltip提示

statinfo (StatInfo)

用于在统计按钮中表示统计信息(常用于form视图右上角)
不可编辑
支持的字段类型: integer, float

  • options
    • label_field: 默认展示字段值 + 字段label， 如果设置了label_field，则展示当前字段值 + label_field字段对应的值

percentage (FieldPercentage)

以百分比的形式展示数据
支持的字段类型: integer, float

  • options
    • show_zero: 是否显示末尾0

percentpie (FieldPercentPie)

以饼图的形式展示 0-100 之间的数字
不可编辑
支持的字段类型: integer, float

progressbar (FieldProgressBar)

以进度条的形式展示一个值
支持的字段类型: integer, float

  • options
    • readonly、editable: 是否只读或可编辑
    • editable_readonly: readonly模式下是否可编辑
    • current_value: 当前值对应的字段(配置该属性，则会取该字段对应的值)
    • max_value: 最大值取该字段对应的值(默认为100)
    • edit_max_value: 最大值是否可编辑(必须配置max_value)，可同步更新max_value对应的字段
    • title: 进度条的标题，会显示在进度条前面

char (FieldChar)

字符型（char）字段的默认字段类型
渲染为单行文本
支持的字段类型: char,text

text (FieldText)

文本型（text）字段的默认字段类型
渲染为多行文本
支持的字段类型: char,text

domain (FieldDomain)

允许用户通过树形结构选择domain条件、实时查看选择记录

reference (FieldReference)

支持的字段类型: char, reference

ace (AceEditor)

提供一个代码编辑器

email (FieldEmail)

渲染为带有锚点的邮件地址
支持的字段类型: char

phone (FieldPhone)

渲染为带有锚点的手机号
支持的字段类型: char

url (UrlWidget)

渲染为带有文本且可点击的锚点，点击新开页签
支持的字段类型: char

  • text: 显示的文本，不配置默认为字段值

link_button (LinkButton)

如果有值，渲染为带有图标且可点击的锚点，点击新开页签
支持的字段类型: char

  • icon: 显示的图标，不配置默认为fa-globe(地球图标)

CopyClipboardChar(TextCopyClipboard)
CopyClipboardText(CharCopyClipboard)

复制粘贴板(单行、多行)
支持的字段类型: char 、text

color (FieldColor)

颜色看板
支持的字段类型: char

binary (FieldBinaryFile)

二进制类型（binary）字段的默认字段类型
允许保存和下载二进制文件
支持的字段类型: binary

  • filename: 二进制文件保存不会保存文件名，因此需要单独的字段保存文件名

image (FieldBinaryImage)

将binary字段显示为实际的图像
支持的字段类型: binary

  • width、height: 图像宽高

  • options

    • preview_image: 实际展示的字段
    • size: [宽, 高](优先级高于width/height)

pdf_viewer (FieldPdfViewer)

pdf预览
支持的字段类型: binary

date (FieldDate)

日期类型(date)字段的默认字段类型
支持的字段类型: date, datetime

  • options
    • datepicker: 日期格式等设置
      • maxDate、minDate: 最大值、最小值（支持固定值与视图中其他字段）
      • format: 日期显示格式

datetime (FieldDateTime)

日期类型(date)字段的默认字段类型
支持的字段类型: date, datetime

  • options 同date

radio (FieldRadio)

以单选组呈现值列表
支持的字段类型: selection、many2one

  • options
    • horizontal: 单选组是否横向排列，默认为false

selection_badge (FieldSelectionBadge)

将值列表渲染为矩形标签
支持的字段类型: selection、many2one

priority (PriorityWidget)

以五角星的形式展示selecttion，星星的数量即为selection值列表的长度减一
支持的字段类型: selection
注意：该挂件在readonly状态下仍然可以编辑

label_selection (LabelSelection)

渲染为一个指定颜色样式的标签
支持的字段类型: selection
不可编辑

  • options
    • classes: 值/样式的字典

state_selection (StateSelectionWidget)

渲染为一个指定的下拉列表
值固定为（normal、done、blocked）
支持的字段类型: selection

kanban_state_selection (StateSelectionWidget)

与 state_selection 完全相同

many2one (FieldMany2One)

多对一（many2one）字段的默认挂件
支持的字段类型: many2one

  • can_create: 允许创建记录，默认为true

  • can_write: 允许编辑相关记录，默认为true

  • options

    • no_create: 禁止创建你新纪录
    • quick_create: 允许快速创建记录，默认为true
    • no_quick_create: 禁止快速创建记录（多余？）
    • no_create_edit: 不允许创建和编辑
    • create_name_field: 创建记录时，输入框显示的字段，默认为name
    • always_reload: 执行额外的name_get获取name值。常用于name_get被重写的情况？？？。默认为false。
    • no_open: readonly模式下，不会重定向到对应记录

m2o_table (m2o_table)

m2o下拉菜单table化组件

  • context

    • table_fields: 表格列字段
    • search_fields: 模糊搜索匹配的字段

many2many_checkboxes (FieldMany2ManyCheckBoxes)

复选框列表
支持的字段类型: many2many

many2many_tags (FieldMany2ManyTags)

以标签列表的形式显示多个值
支持的字段类型: many2many

  • options
    • color_field: 颜色字段（int）

many2many_binary_print (FieldMany2ManyBinaryMultiFiles_Print)
many2many_binary (FieldMany2ManyBinaryMultiFiles)

附件的展示和预览
支持的字段类型: many2many

自定义挂件

many2one

  • widget="img_item" : many2one图标选择

  • options="{'select_all': 1}"”

  • widget="o2m_unique": 用于one2many中的many2one字段，使得one2many行数据不会重复

  • widget="only_create": 点击many2one字段弹出创建框

one2many

  • dy_sequence: 字段默认序列

  • fixed_col: 固定列

  • widget="multi_select_tree": 明细行批量添加挂件

  • widget="one2many_form": 以form形式新建行记录

  • not_creatable、not_editable、not_deletable 行动态配置

many2many

  • widget="many2many_picscroll" : many2many字段图片视频轮播
  • widget="many2many_binary_print": 文件预览

开发规范

发表于 2021-09-27 分类于 odoo12

开发规范

  • 命名规范

    模块名统一采用 模块名.文件名。文件名由具体功能决定

  • 源码继承

    文件结构与源码保持一致（主要针对dy_base），命名与源码保持一致（只更换模块名）

  • 文件功能单一

    单个文件的功能尽量单一，尽量避免将多个功能揉在一个文件中

  • 文件根据类型放在相应的文件夹

    • js ===> js文件夹
    • css ===> css文件夹
    • scss ===> scss文件夹
    • 第三方资源 ===> lib文件夹

  • 代码格式化

    提交代码之前，格式化代码（pycharm默认格式化即可）

  • 报表格式统一

    • 后续专门整理统一报表开发规范的方案
